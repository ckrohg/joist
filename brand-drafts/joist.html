[ 01 / why this exists ]

Every prior AI builder shipped the same bugs.

We read every public bug tracker, postmortem, and Trustpilot review of Wix AI, 10Web, Lovable, Elementor MCP servers, Builder.ai. Same failure modes everywhere. Joist's architecture refuses each one by name.

silent_write_success
Tool returned {success: true} while the actual save to _elementor_data never landed. Page rendered broken; the agent thought it worked.
schema_drift
LLM emitted justify_content instead of flex_justify_content. Plugin accepted it. Frontend rendered default alignment. (msrbuilds #32, the canonical case.)
regenerate_destroys
User asked to "redesign the hero." AI regenerated the entire page and deleted three weeks of human work.
cache_drift
Write succeeded in the DB. Cache served the old version for hours. Agent and human disagreed on what was live.
round_trip_clobber
Human opened Elementor between two agent calls. Agent's next write overwrote the human edits. No conflict detection.
credit_burn_loop
Agent looped on its own bug. Spent $40 of Anthropic credits proposing six wrong fixes to the same problem.

Joist refuses to do any of these. We treat each as a hard invariant. 30 total. Read all 30 →

[ 02 / the write pipeline ]

Every write goes through one path.

DocumentWriter::save() is the spine. It enforces 9 of the 30 invariants in a single method. update_post_meta('_elementor_data', …) is grep-banned in the codebase. There is no second way to write.

01 GATE
PolicyGuard
Refuse-list runs first. No force-delete. No arbitrary zip install. No agent role doing user CRUD.
02 VALIDATE
Live schema
Every widget setting checked against introspected schema. Unknown keys → 422 with Levenshtein-1 + flex_*-aware suggestion.
03 SNAPSHOT
Atomic revision
Custom revisions table. Gzipped LONGBLOB. Restore is one call. WP's built-in revisions don't cover postmeta — ours do.
04 WRITE
Document::save()
Through Elementor's own path. Slash handling, hooks, version stamping identical to a human edit.
05 VERIFY
Read-after-write
Tool returns the post-save element + new hash. Never {success: true}. CSS regen + cache flush deferred async.
// The actual spine. (Simplified — see specs/ARCHITECTURE.md §4)

public function save(SaveRequest $req): SaveResult
{
    $this->policy->assertAllowed($req);             // constraint #18
    $this->opMode->intercept($req);                // observer/quiet/kill
    $this->sessions->recordOp($req);                 // chained-singleton (#19)

    $lock = $this->locks->acquire($req->postId);
    try {
        $this->elementor->assertSupportedVersion();
        $this->layoutMode->validateInserts($req);     // container vs section (#23)
        $this->dynamicTags->validateTree($req);      // no dangling refs (#25)
        $this->validator->validateTree($req);         // schema + responsive (#1, #24)

        [$elements, $tx] = $this->globals->preferGlobals($req); // (#26)
        $revId = $this->revisions->snapshot($req);

        $doc = $this->elementor->getDocument($req->postId);
        $doc->save(['elements' => $elements]); // Elementor's own path

        $verified = $doc->get_elements_data();        // read-after-write (#2)
        $this->audit->log('document.save', $req, $verified); // chain_hash (#30)

        wp_schedule_single_event(time() + 1, 'joist_post_save_verify', [
            'post_id' => $req->postId,                  // async: regen CSS, flush cache,
            'hash'    => $this->hasher->forElements($verified), // verify frontend, fire webhook
        ]);

        return new SaveResult($verified, $tx, $revId);
    } catch (\Throwable $e) {
        $this->revisions->rollback($revId);              // no silent failure (#16)
        throw $e;
    } finally {
        $this->locks->release($lock);
    }
}
[ 03 / what makes joist different ]

The discipline is the product.

Four invariants that took every prior tool a year of bug reports to discover. Joist enforces them on every write, by design.

[ 01 ]
Validate against the live site.

Every widget setting checked against the introspected schema of that specific install — not a hardcoded one. Unknown keys never pass through silently. Suggestions include Levenshtein-1 + flex_*-aware fuzzy matches so the agent self-corrects in one round-trip.

// agent writes: { "justify_content": "center" } // joist responds 422 with: { "code": "schema.unknown_key", "recovery_suggestions": [ { "op": "update_settings", "args": { "flex_justify_content": "center" }, "rationale": "Levenshtein-1 + flex_*-aware match" } ] }
[ 02 ]
Plan Mode is a real plan.

The agent proposes, you approve in WP admin with a side-by-side preview, the executor runs with atomic rollback. Single-call patches are gated by a chained-singleton detector — you can't bypass the plan by issuing N micro-edits.

[ 03 ]
Round-trip safe.

Every read returns a content hash. Every write echoes the hash it was planned against. Human edits in the Elementor UI invalidate that hash — the agent's next write fails fast and re-reads. No clobbering. No "the AI deleted my work" tickets.

[ 04 ]
Refuses to ship slop.

Anti-slop rules in the codebase reject indigo-500 gradients, "Build the future of X" headlines, AI 3D blobs, uniform 16px radii. Steers toward shadcn / Aceternity / Awwwards taste sources — not Envato kits.
