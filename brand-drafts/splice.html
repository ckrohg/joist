[ 01 ] What Splice refuses

Six bugs killed every prior AI builder. We engineered them out.

We read every public bug report. Wix AI, 10Web, Lovable, the half-dozen open-source Elementor MCP servers, Builder.ai. Same failure modes everywhere. Below: each one, and the specific Splice invariant that makes it structurally impossible.

01
No silent writes
Tool returned {success:true} while the save never landed. We require read-after-write on every mutation. Verified element returned, never just a success flag.
02
No schema drift
LLM wrote justify_content when Elementor expected flex_justify_content. Unknown keys 422 instantly with Levenshtein + flex_* suggestions.
03
No regenerate-clobber
"Redesign my homepage" should not delete three weeks of work. Surgical edits by element ID only. Full regenerate as a single tool call: refused.
04
No cache drift
Write succeeded in DB. Cache served the old version for hours. We flush Elementor CSS + page cache + CDN after every write, then verify the frontend before resolving.
05
No round-trip clobber
Optimistic concurrency via content hashes. Human edits between agent calls invalidate the hash — agent's next write fails fast and re-reads. No mystery overwrites.
06
No credit-burn loops
Per-task spending cap with cheap-mode fallback. After N retries on the same error class, escalate to human. No $40 spent debugging the agent's own bug.
[ 02 ] The write pipeline

One path. Five gates. Async verify.

DocumentWriter::save() is the spine. Every write funnels through it. update_post_meta('_elementor_data', …) is grep-banned in the codebase. There is no second way to write — by design.

// the only write path~200ms sync · verify async
01
Guard
PolicyGuard refusals + chained-singleton plan trigger before any work.
02
Validate
Schema check against the site's live widget registry. Unknown → 422 + suggestion.
03
Snapshot
Atomic revision into custom table. Restore is one call. Rollback on any failure.
04
Write
Through Elementor's own Document::save(). Slash handling, hooks, version stamping native.
05
Verify
Read-after-write returns post-save element + new hash. CSS regen + cache flush deferred.
