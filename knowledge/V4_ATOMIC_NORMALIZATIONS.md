# V4 Atomic Normalizations (Elementor 4.0.x)

This file documents Elementor V4's atomic transformer normalizations as observed via Joist's read-after-write hash defense. It is the source of truth for `Hasher::V4_AUTO_FIELDS` and the related lenient-equality rule in `normalizeNode()`.

If a write trips `atomic_save_silent_failure` with `failure_reason: shape_divergence_check_b` and the `structural_diff.full_tree_sample` reveals a *new* V4-added field, **extend `V4_AUTO_FIELDS` here and in `plugin/src/Core/Hasher.php`, then re-deploy.** Iterate until the trip stops.

## Confirmed V4 auto-added fields (as of 2026-05-31)

| Field | Where added | Default | Strip rule |
|---|---|---|---|
| `isInner` | All containers + atomic structural elements | `false` | always |
| `id` | All elements | regenerated by Elementor id-manager | always |
| `styles` | Atomic elements (e-*) | `[]` | always |
| `interactions` | Atomic elements (e-*) | `[]` | always |
| `editor_settings` | Atomic elements (e-*) | `[]` | always |
| `version` | Atomic elements (e-*) | `"0.0"` | always |
| `elements` | Widget nodes (V3 + atomic widgets) | `[]` | only when value is `[]` (preserve non-empty children) |

## Production shape decision

**V3-on-V4 is the production-correct path** for content-heavy plans on Elementor 4.0.x as currently shipped. Reasoning:

1. The only atomic elements registered on Elementor 4.0.9 + JupiterX are **structural** (e-flexbox, e-div-block, e-tabs*, e-form*). There are **no atomic content widgets** (no e-heading, e-paragraph, e-button, e-image) registered.
2. V4 e-flexbox + V3 widgets nests cleanly (proved 2026-05-31, plan `pln_MPTBRPMG...`, page 148, revision 23), but e-flexbox styling requires the atomic prop system (Classes, Variables) which Joist does not yet author.
3. V3 `container` with explicit `padding` + `content_width` renders identically to legacy V3 sites and round-trips cleanly through V4 with the auto-fields stripped.

**CloneGenerator and PlanGenerator emit V3 shapes (`container`, `heading`, `text-editor`, etc.) regardless of site Elementor version.** A future Wave (post v1) will revisit when atomic content widgets register upstream.

## How to verify after any deploy

Run `joist_smoke_test_roundtrip` via MCP. It exercises both V3 container and V4 e-flexbox hybrid shapes against the live site and reports pass/fail per shape with the structural_diff if anything trips.
